# fix-a419079938b8cf63 — E2T1 and E2T2 bypass toolkit storage with isolated direct-Redis connections

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TALLY

`src/handlers/E2T1.ts` and `src/handlers/E2T2.ts` each create their own ioredis client via `createRequire("ioredis")` instead of using the toolkit's `resolveSessionStorage`. This is the same anti-pattern already filed for E1T4 in `fix-808f541dfe2f1a55`, but E2T1 and E2T2 are NOT covered by that fix task.

- E2T1 writes to `tallybot:{userId}:counters` hash (field `default`).
- E2T2 writes to `e2t2:{key}` hash (field `value`).

Neither integrates with `counterStore` or `stores/counter.ts`. E2T2 in particular uses `maxRetriesPerRequest: null` which makes Redis errors silently queue rather than failing — this can mask connection failures.
